Privacy Risks of General-Purpose Language Models论文阅读
更新于 2020-09-20 论文阅读

论文题目：Privacy Risks of General-Purpose Language Models

论文原作者：Xudong Pan、Mi Zhang、Shouling Ji、Min Yang

原文链接：https://ieeexplore.ieee.org/document/9152761/references#references

该篇论文是2020年信息安全四大顶会之一的S&P会议接收的论文。该篇论文对目前最新的8种语言模型的隐形风险首次进行了研究，并且构建了两种攻击类别：模式重建攻击和关键字推断攻击，证明了目前通用语言模型存在泄漏敏感数据的风险，攻击者可能会对模型中的文本嵌入进行逆向工程，以获取文本中的敏感信息。

该篇论文的研究内容非常新颖，在预训练语言模型给众多NLP下游应用带来性能提升的同时，这篇论文从隐私泄露角度出发，反其道而研究语言模型中的文本嵌入的隐私泄露风险，个人感觉非常值得一读。

摘要译文

最近，在自然语言处理(NLP)中为文本特征提取构建通用语言模型的新范例(例如Google的BERT和OpenAI的GPT-2)已经出现，这是自然语言处理系统中将文本转换为向量(即embedding，嵌入)以进行下游建模的标准过程，并开始在各种下游NLP任务和现实世界系统(例如Google的搜索引擎[6])中找到应用。 为了获得通用的文本嵌入，这些语言模型具有高度复杂的结构，具有数百万个可学习参数，并且在使用之前通常要对数十亿个句子进行预训练。 正如广泛认识的那样，这样的实践确实提高了许多下游NLP任务的最先进性能。

但是，改进后的实用程序不是免费的。 我们发现，通用语言模型中的文本嵌入会从纯文本中捕获很多敏感信息。一旦被对手访问，可以对嵌入进行反向工程，以披露受害者的敏感信息，以进一步骚扰。 尽管这样的隐私风险可能会对这些有前途的NLP工具的未来使用构成一定的威胁，但到目前为止，主流行业级语言模型都没有公开的攻击或系统的评估。

为了弥合这种差距，我们通过四个案例研究，对8种最新语言模型的隐私风险进行了首次系统研究。 通过构建2种新颖的攻击类别，我们的研究表明上述隐私风险确实存在，并且可能对通用语言模型在身份，基因组，医疗保健和位置等敏感数据上的应用构成实际威胁。 例如，当从病人的医学描述的Bert嵌入中推断出精确的疾病部位时，我们显示出该对手几乎没有先验知识可以达到约75％的准确度。 作为可能的对策，我们提出了四种不同的防御措施（通过舍入，差分隐私，对抗训练和子空间投影）来模糊化未保护的嵌入，以达到缓解目的。 通过广泛的评估，我们还对每种防御措施带来的效用-隐私权衡进行了初步分析，我们希望这可以促进未来的缓解研究。

论文贡献

  • 发现了通用语言模型中潜在的隐私风险，结果表明，几乎为零的对手可以访问文本嵌入，从而可以泄露未知文本中的敏感信息。
  • 设计了一条通用的攻击流程来利用文本嵌入中的用户隐私，并使用先进的深度学习技术实施两次实际攻击，以证明存在隐私风险。
  • 对8种最先进的通用语言模型进行首次系统评估，其中包括4种不同的案例研究，以证明隐藏的隐私风险，并对影响隐私的因素进行深入分析。
  • 提供了对四种可能的对策及其效用与隐私权衡的初步研究，我们希望这些对策可以促进未来的防御研究。

embedding

论文中定义句子嵌入为：给定一个词汇表$V$，一个序列$x:=(w_1,…,w_n)$是长度为$n$的句子，如果句子中的每一个词$w_i$或token都在词汇表中。特征抽取器$f$为句子到实向量空间中的映射，对于句子$x$，向量$z:=f(x)$称为句子的嵌入。

在通用语言模型之前，word2vec、TF-IDF等模型均可助力词向量任务。近年来通用语言模型的发展，改变了人们对于嵌入的理解和使用。我们可以通过这些模型获取具有更好通用性的句子嵌入，将其直接输入到下游的训练模型。

  1. 根据准备好的词汇进行标记化（tokenization）。
  2. token embedding，即借助可学习的查找表将token映射到相应的向量。
  3. 沿两个维度通过Transformers传播，在最后一层将句子转化为长度为n的向量序列，即隐藏状态。
  4. 对隐藏状态实施池化操作，以得到句子嵌入。

通用攻击流程

论文中将攻击流程分为四个阶段。如下图所示：

1、准备外部语料库

论文中主要给攻击者建议了两种方式来准备外部语料库：

  • 通过生成算法来创建外部语料库；
  • 在公开领域的公共预料库中生成。然后对于语料库中的每个x应用算法$P$得到标签$P(x_i)$。

2、查询语言模型

该阶段是将Dext中的句子转化为相应的嵌入，得到训练集
$
D_{train}:=\left { \left ( z_i,P\left ( x_i \right ) \right ) \right }_{N}^{i=1}
$
其中$z_i$是与句子$x_i$对应的嵌入。

3、训练攻击模型

通常将模型训练为分类器，即以句子嵌入$z_i$为输入，输出所有敏感信息可能值的概率向量$g(z_i)$。根据攻击者的不同知识水平，分类器的选择也会不同。

4、推断

训练完模型后，给定目标句子的嵌入$z$，攻击者会根据以下的等式推断敏感信息$s:=A\left ( z \right )=argmax x_{i\in \left { 1,2,…,K \right }}\left [ g\left ( z \right ) \right ]_i$

模式重建攻击

论文中将模式重建攻击定义为恢复具有固定格式的纯文本的特定段，例如身份证号、基因组序列中的某些特定片段可能会含有敏感信等。在该部分，论文作者假定攻击者了解文本的生成规则，并且可以使用生成算法来准备外部语料库。

论文中以中国大陆的身份证号和基因组序列作为实例来进行攻击实验。

1、身份证号

论文中根据下列等式规则随机生成1000个身份证号作为受害者：
$$P_{citizen}:|residence|birthday|extra|\rightarrow |birthday| $$

在实施攻击时，作者针对身份证号中的生日日期进行模拟攻击。为了降低参数，将攻击分为年、月、日三个子攻击，使用三层MLP实现，分别包含400、25、200个具有Sigmoid激活函数的隐藏单元。每个子攻击的训练批次大小设置为128。

对于实验结果，论文中给出了子攻击的Top-1和Top-5的准确率，和经过十万次迭代训练后通过整体攻击推断出整个出生日期的情况。结果如下表所示:

从表格中可以看出，模拟攻击能够有效地从身份证号嵌入中恢复敏感信息。当给定Transformer-XL作为语言模型嵌入身份证号时，模拟攻击能够以超过80%的Top-1的准确率恢复受害者的确切生日月份和日期，以超过62%的Top-5准确率恢复整个出生日期。

2、基因组序列

针对基因组的实验，论文中使用HS3D公共基因数据集，用于基于剪接位点预测。所有基因组序列的长度为20。文中假设测试集中的基因组序列的嵌入被泄露给了攻击者，其中包含分别具有或不具有剪接位点的1000个样本。

模拟攻击模型使用了4层的MLP。攻击者输入受害者的基因组嵌入和目标位置，模型输出预测的核苷酸类型。

实验结果如下图所示：

当给定基因组序列的GPT嵌入时，模拟攻击实现了在两端推断受害者核苷酸类型的接近100%的准确率，平均超过62%的准确率。

通过身份证号和基因序列两组实验，论文中发现了RoBERT模型在两种情况下均显示出比其他语言模型更强的鲁棒性。RoBERT与BERT相比使用了以字节为单位对句子进行标记化，而不是字符或者单词。并且RoBERT在面对相同的攻击时显示的隐私风险比BERT约低50%，因此论文推测是因为字符级标记化方案使句子嵌入中的敏感信息不那么明显。

同时，论文中还发现了大多数精度曲线程谷状形状，意味着大多数语言模型更能捕获句子头尾的标记信息，而不是中间，位于句子边界的敏感信息更容易被泄露。

关键字推断攻击

除了对特定结构的文本进行研究，论文中还以任意自然句子为文本进行研究。与模式重建攻击不同，关键字推断攻击会探究某些关键字的出现而不是整个序列的精确重构。

论文中以航空公司评论和医学描述两个实例进行了模拟攻击实验。

1、航空公司评论

航空公司借助NLP技术处理大量文本形式的航空公司评论，以了解客户的观点。攻击者一旦访问到文本的嵌入，就可以推断出与受害者有关的各种与位置相关的敏感信息。论文中针对评论文本中包括特定城市名称的可能性进行了研究。

在模拟攻击时，论文中分为了白盒和黑盒，即假设攻击者有文本的信息和没有文本的信息。

在白盒中，攻击者有一个与未知纯文本相同的分布中抽取的语料库作为外部语料库抽取标签。为了平衡标签，实验时，作者进行了单词替换。

在黑盒中，作者建议攻击者使用公共语料库创建外部语料库，然后将攻击模型在外部语料库上的对抗性知识动态地传递到目标语料库。

下图是论文中的黑盒攻击模型体系结构：

实验数据集是从Skytrax收集的航空公司评论，并保留包含10个指定城市名称之一的评论，以形成基准数据集，其中包含4685条航空公司评论。文中将数据集按照10:1的比例分为测试集和假设攻击者已知的数据集，用于模拟白盒设置。在白盒设置下，论文中研究了攻击模型为线性SVM和具有80个隐含单元的3层Sigmoid激活的MLP两种实现。黑盒设置中的外部语料库是从Yelp-Food数据集中收集的。黑盒设置下的攻击模型为线性SVM、3层MLP和DANN三种实现。

论文中报告了基于DANN攻击的针对每个关键字的准确性，以及不同攻击模型的白盒和黑盒设置中平均10个关键字推理攻击的性能，如下图所示：

当给定GPT和GPT-2嵌入时，白盒攻击和黑盒攻击在推断特定城市名称上平均达到了95%和75%以上的准确率。白盒攻击中，XL和RoBERTa的鲁棒性比其他语言模型强得多。

2、医学描述

在医学领域，论文假设攻击者通过推断医学描述文本中与身体有关的单词的出现概率来确定受害者的确切疾病部位。在实验设置中也分成了黑盒设置和白盒设置，使用了与航空公司评论相同的攻击模型。数据集使用了CMS公共医疗记录，并且与航空公司评论实验设置一样，将数据集进行10:1的随机分割。关键字为包含在数据集中10个与身体有关的词。

下图所示，为在医学描述数据集上进行黑盒和白盒攻击的实验结果：

从图中可以看出，在白盒攻击中，在推断特定身体部位时达到了99%以上的准确率；而黑盒攻击在没有先验知识的情况下，平均仍然达到75%以上的准确率。

下表显示了在5个随机关键字的推断中，基于DANN的攻击在这两个实例中的平均和最坏情况。黑盒攻击在大多数情况下达到了90%以上的平均准确率。

防御建议

论文在遵循对嵌入进行模糊处理的原则上，评估了四个可能的技术选择以尽可能消除文本嵌入中敏感信息的泄露。

  1. 舍入：对文本嵌入的每个坐标上应用浮点舍入以进行模糊处理。
  2. Laplace机制：使用来自Laplace分布的样本来干扰嵌入坐标
  3. 隐私保护映射：以对抗性训练为基础。
  4. 子空间投影：从通用句子嵌入空间中投影出子空间对关键字出现编码。

论文中评估了前三个防御措施，结果如下所示:

论文表示,每种防御都可以将攻击者削弱为一个完全随机的猜测着，但同时会对下游任务的效用有所损害。

阅读感想

这篇论文花了不少的时间去阅读，虽然了解了论文的大体工作，但对于论文中的一些实现细节还有一些读不懂的地方，可能在理解上也存在一些错误。针对附录部分的一些扩展内容，我也没有仔细去看，

这篇论文最吸引我的地方是它的思路非常的新颖，一定程度上给人们在通用语言模型的使用上带来了思考。同时作为一篇会议论文，在前面部分对于NLP中预训练模型和embedding的描述也非常精简准确，能够使人有个快速了解。在论文的实验部分中考虑的很周全，并且作者将一些细节放在附录部分，让读者选择阅读。

总之这篇论文推荐给大家。
